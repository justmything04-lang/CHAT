================================================================
  GOOGLE SHEET ARCHITECTURE (7 TABS)
================================================================

Tab 1  DASHBOARD    — Visual. Auto-rebuilt by bot on every update.
                       Metric cards: progress%, streak, days left, status
                       Colour-coded this-week plan (green=done, blue=today)
                       Weak areas table + resources links
                       Bot exports Tab 1 as PNG → sends on Telegram (/progress)

Tab 2  STUDY_PLAN   — AI-generated day-by-day plan. Bot reads/writes.
Tab 3  PROGRESS_LOG — Every time student reports progress.
Tab 4  TEST_RESULTS — Quiz scores, weak questions, retest schedule.
Tab 5  WEAK_AREAS   — Topics flagged as difficult, priority, resources.
Tab 6  RESOURCES    — Gamma slide links, NotebookLM, research links.
Tab 7  SESSION      — Meta (used only by GSheet storage backend).

================================================================
  ALL BOT COMMANDS (18 total)
================================================================

Study management:
  /start       — onboarding (name → goal → date → plan creation)
  /today       — today's targets from sheet
  /done        — mark topic complete
  /stuck       — flag weak area
  /progress    — report + Tab 1 snapshot sent as photo
  /reschedule  — AI reshuffles remaining plan
  /report      — weekly summary

AI tools:
  /test        — 5-question MCQ quiz, auto-scored, sheet updated
  /research    — Sparkpage-style structured summary (Genspark replacement)
  /explain     — plain-language explanation ≤150 words
  /compare     — side-by-side concept comparison
  /mnemonic    — memory tricks / acronyms
  /slides      — Gemini outline → Gamma deep link (one-click slide deck)

Material Q&A:
  /upload      — PDF → NotebookLM notebook (Gemini fallback if unavailable)
  /ask         — query uploaded material
  /podcast     — audio overview MP3 (NotebookLM only)

Settings:
  /settings    — inline keyboard: change briefing time, alert frequency, reset
  /help        — full command list

Natural language also works for all common actions.

================================================================
  CONFIGURABLE BY USER FROM TELEGRAM (inline keyboard)
================================================================

  Briefing time:     any HH:MM  (default 08:00, per user's timezone)
  Inactivity alert:  Hourly / Daily / Every 2 Days / Weekly / Never
  Reset all data:    with confirmation step
  View sheet:        direct link to their Google Sheet

================================================================
  STORAGE BACKENDS
================================================================

  sqlite    — default, zero-config. ⚠️ resets on Render redeploy.
  supabase  — production. Free 500 MB. Run supabase_setup.sql first.
  gsheet    — educational. Uses a Google Sheet as flat DB.

  Switch via: STORAGE_BACKEND=sqlite|supabase|gsheet  in .env

================================================================
  DEPLOYMENT — RENDER
================================================================

  1. Push to GitHub
  2. Render → New Web Service → connect repo
  3. Root: ai_study_partner
  4. Build: pip install -r requirements.txt
  5. Start: python main.py
  6. Add all env vars from .env.example
  7. Set RENDER_APP_URL = your Render URL
  8. Recommended: STORAGE_BACKEND=supabase (persistent across deploys)

  Bot auto-detects: RENDER_APP_URL set → webhook mode
                    not set → polling mode (local dev)
